# Шаблон акции

Настраивается аналогично акции в разделе Алгоритм настройки акции

# Шаблон Email

Для настройки шаблона email создайте новый шаблон и заполните следующие поля:

  • Тема письма – заголовок письма.
  • Письмо – непосредственно текст письма. Для добавления HTML-верстки нажмите на кнопку <> и вставьте HTML-код.
  • Метапеременные

WARNING

При добавлении метапеременной в текст подставляется конструкция вида [%anketa.Contact.PersonalData.LastName%]. При отправке письма пользователю эта конструкция будет заменена на соответствующий ей текст.

WARNING

Если тема письма не заполнена, оно не будет отправлено клиенту.

# Шаблон СМС-сообщения

Для настройки шаблона смс создайте новый шаблон и заполните следующие поля:

  • Письмо – непосредственно текст СМС-сообщения.
  • Метапеременные

WARNING

При добавлении метапеременной в текст подставляется конструкция вида [%anketa.Contact.PersonalData.LastName%]. При отправке письма пользователю эта конструкция будет заменена на соответствующий ей текст.

# Шаблон Viber-сообщения

Для настройки шаблона viber создайте новый шаблон и заполните следующие поля:

  • URL-ссылка на изображение – абсолютная ссылка на изображение, которое должно отображаться в сообщении. Если изображение не нужно - оставить поле пустым.
  • Письмо – непосредственно текст сообщения.
  • Текст кнопки – текст кнопки, который должен отображаться в сообщении на кнопке.
  • URL-ссылка кнопки – ссылка, по которой должен перейти пользователь после нажатия на нее в сообщении.
  • Метапеременные

WARNING

При добавлении метапеременной в текст подставляется конструкция вида [%anketa.Contact.PersonalData.LastName%]. При отправке письма пользователю эта конструкция будет заменена на соответствующий ей текст.

WARNING

Если Кнопка в сообщении не нужна, то поля Текст кнопки и URL-ссылка кнопки должны остаться пустыми. Если из них хотя бы одно поле не заполнено, то сообщение не будет отправлено пользователю.

# Шаблон Push. Кошелёк

Для настройки шаблона Push. Кошелёк создайте новый шаблон и заполните следующие поля:

  • Заголовок Push-сообщения – тема Push-сообщения.
  • URL-ссылка на изображение – абсолютная ссылка на изображение, которое должно отображаться в сообщении. Если изображение не нужно - оставить поле пустым.
  • Текст Push-сообщения – непосредственно текст сообщения.
  • Куда пользователь должен перейти по клику на Push–сообщение – ссылка, по которой должен перейти пользователь после нажатия на нее в сообщении.
  • Метапеременные

WARNING

При добавлении метапеременной в текст подставляется конструкция вида [%anketa.Contact.PersonalData.LastName%]. При отправке письма пользователю эта конструкция будет заменена на соответствующий ей текст.
